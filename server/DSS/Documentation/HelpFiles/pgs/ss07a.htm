Setting up authentication

    Darwin Streaming Server comes with an authentication module you can use to control access to streamed media files.

    • QTSSAccessModule
    This module lets you set up access to Darwin streaming services, but not to any other services on your computer.

    Users must have QuickTime Player 4.1 or later to access a media file for which authentication is enabled. The user must enter his or her user name and password to view the media file. Users who try to access a media file with an earlier version of QuickTime installed will see an error message: "401: Unauthorized."

    Note: Authentication modules control access to media streamed from your server. It does not control access to media you relay. Access to media streamed from a relay server must be set up by the administrator of the relay server.

Enabling or disabling an authentication module

    To enable an authentication module, copy it to the Darwin Streaming Server modules directory, which is located at /usr/sbin/StreamingServerModules.

    To disable an authentication module, remove it from the modules directory, and copy it to the /usr/sbin/StreamingServerModules.disabled directory.

    When you install Darwin Streaming Server, QTSSAccessModule is enabled by default.

Setting up an authentication module

    If you are using QTSSAccessModule, set up an access file and user and group files.
