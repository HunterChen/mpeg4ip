Viewing access and error logs

    If logging is turned on in the configuration file for the Darwin Streaming Server, information about access and errors is saved to log files.

    To open a log:
    1 Look in /var/streaming/logs
    2 Open either the StreamingServer log or the Error log.

Updating and saving logs

    Each week the current log file is renamed with the current date and a new log file is started.

    To save the information in the current log file at any time:
    • Open the log file and save it with a different name.
    A new log file is started.

Analyzing the access log

    The information that's stored in the access log can be analyzed using third-party software. See Access log format for a list of information in the log.
